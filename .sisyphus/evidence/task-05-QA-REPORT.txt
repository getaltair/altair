================================================================================
                              BUG FIXES APPLIED
================================================================================

Issue: Database error on create relation (500)
Root Cause: PostgreSQL custom enum types require explicit casting when
            binding text parameters in prepared statements.

Fix: Added explicit type casts to all SQL queries:
  - INSERT VALUES: $1::entity_type, $3::entity_type, etc.
  - WHERE clauses: from_entity_type = $1::entity_type
  - RETURNING: from_entity_type::text (to get text for Rust model)
