Root cause follow-up
- Existing `web-db-1` container from `apps/web/compose.yaml` was using host port 5432 and had to be stopped temporarily for this QA run.
- `docker compose logs powersync` shows the pinned `journeyapps/powersync-service:0.2.0` image rejects `storage.type: postgresql` with `TransformError: Expected 'mongodb' but got 'postgresql'`.
- `docker compose logs postgres` also shows `CREATE PUBLICATION IF NOT EXISTS powersync FOR ALL TABLES;` fails on PostgreSQL 16 with `syntax error at or near "NOT"` during init.
