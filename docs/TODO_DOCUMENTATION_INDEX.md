## Writing Guidelines for All TODO Docs

### ADHD-Friendly Format Requirements

All documentation must follow these principles:

**Structure:**
- ✅ Short paragraphs (2-3 sentences max)
- ✅ Numbered steps for procedures
- ✅ Bullet points for lists
- ✅ Headers for scannability
- ✅ TL;DR at the top

**Visual Aids:**
- ✅ Screenshots for every major step
- ✅ GIFs for workflows
- ✅ Diagrams for concepts
- ✅ Code examples with syntax highlighting
- ✅ Emoji for visual anchors (don't overdo it)

**Tone:**
- ✅ Encouraging without patronizing
- ✅ Assume intelligence, not prior knowledge
- ✅ Celebrate small wins
- ✅ Normalize ADHD struggles
- ✅ No jargon (or explain it)

**Accessibility:**
- ✅ Alt text for all images
- ✅ Descriptive link text (not "click here")
- ✅ Proper heading hierarchy
- ✅ Works with screen readers
- ✅ Mobile-friendly
