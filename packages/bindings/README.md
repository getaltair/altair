# @altair/bindings

Generated TypeScript bindings for Tauri IPC commands.

## Overview

This package contains TypeScript type definitions for all Tauri commands defined in the Rust backend. The types are automatically generated using [tauri-specta](https://github.com/specta-rs/tauri-specta) to ensure type safety between the Rust backend and TypeScript frontend.

## Current Status

**Phase 3 (placeholder)**: This package currently exports placeholder types that mirror the expected domain model. These placeholders allow frontend development to begin before the Tauri apps are scaffolded.

**Phase 4 (generated)**: Once the Tauri apps are set up (task 4.5), running the build will generate actual bindings from Rust type definitions into `src/generated.ts`.

## How Bindings Are Generated

### 1. Rust Side (backend)

Tauri commands are defined in `backend/crates/altair-commands/` with the `#[tauri::command]` and `#[specta::specta]` attributes:

```rust
use specta::Type;
use tauri_specta::Event;

#[derive(Type, serde::Serialize)]
pub struct Quest {
    pub id: String,
    pub title: String,
    pub energy_cost: EnergyCost,
    // ...
}

#[tauri::command]
#[specta::specta]
pub async fn create_quest(
    state: State<'_, AppState>,
    title: String,
    energy_cost: EnergyCost,
) -> Result<Quest, ApiError> {
    // ...
}
```

### 2. Build Script (Tauri app)

Each Tauri app's `src-tauri/build.rs` runs tauri-specta to generate bindings:

```rust
// apps/guidance/src-tauri/build.rs
fn main() {
    let builder = tauri_specta::Builder::<tauri::Wry>::new()
        .commands(tauri_specta::collect_commands![
            altair_commands::quest::create_quest,
            altair_commands::quest::list_quests,
            // ...
        ]);

    builder
        .export(Typescript::default(), "../../../packages/bindings/src/generated.ts")
        .expect("Failed to export typescript bindings");

    tauri_build::build();
}
```

### 3. Generated Output

The build produces `src/generated.ts` with:

- Type definitions for all Rust structs
- Type-safe `invoke()` function
- Event type definitions

```typescript
// Auto-generated by tauri-specta
export const commands = {
  createQuest: (title: string, energyCost: EnergyCost) =>
    invoke<Quest>('create_quest', { title, energyCost }),
  listQuests: () => invoke<Quest[]>('list_quests'),
  // ...
};
```

## Usage

### In Frontend Code

```typescript
import { commands, type Quest } from '@altair/bindings';

// Type-safe command invocation
const quest: Quest = await commands.createQuest('Do the thing', 'medium');

// Type checking catches errors at compile time
await commands.createQuest(123, 'invalid'); // TS Error!
```

### Importing Types Only

```typescript
import type { Quest, EnergyCost, Note } from '@altair/bindings';
```

## Development

### Regenerating Bindings

After modifying Rust commands or types:

```bash
# From repo root
pnpm --filter guidance build  # or any Tauri app

# Bindings are updated at packages/bindings/src/generated.ts
```

### Local Development

The placeholder types in `src/index.ts` allow frontend development before bindings are generated:

```bash
pnpm --filter @altair/bindings build  # Compiles placeholder types
pnpm --filter @altair/bindings dev    # Watch mode
```

## Files

| File               | Purpose                                  |
