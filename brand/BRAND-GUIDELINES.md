# Altair Brand Guidelines

**Version 1.0** • Updated October 2025

---

## Brand Overview

Altair is an ADHD-friendly project management platform that helps neurodivergent individuals navigate their tasks and projects. Our brand reflects clarity, guidance, and the bright star that gives us our name.

### Brand Personality

- **Clear & Direct** - No ambiguity, no confusion
- **Supportive & Understanding** - Built by and for the ADHD community
- **Professional yet Approachable** - Serious about helping, not taking ourselves too seriously
- **Modern & Technical** - Cutting-edge tech with thoughtful design

---

## Logo & Icon

### Primary Icon: Diamond Star

The Altair icon is a geometric diamond-shaped star representing the brightest star in the Aquila constellation. It symbolizes navigation, guidance, and finding your way through complexity.

**Icon Specifications:**
- **Format:** SVG (preferred) or PNG
- **Sizes:** 512px, 256px, 128px, 64px
- **Safe area:** 10% padding on all sides
- **Background:** Dark circle (#0F172A) with 45px radius

### Usage Rules

✅ **DO:**
- Use adequate whitespace around the icon (minimum 10% of icon size)
- Maintain the aspect ratio when resizing
- Use on contrasting backgrounds
- Pair with Inter or similar sans-serif fonts
- Use the SVG version whenever possible

❌ **DON'T:**
- Rotate, skew, or distort the icon
- Add drop shadows, outlines, or other effects
- Change the gradient colors
- Place on busy or low-contrast backgrounds
- Stretch or compress disproportionately

### Icon Variations

1. **Full Color** (Primary) - Use in most contexts
2. **White** - For dark backgrounds or monochrome applications
3. **Black** - For light backgrounds (rare use)

## Typography

### Primary Font: Inter

**Why Inter?**
- Excellent readability at all sizes
- Large x-height for accessibility
- Wide availability
- Professional yet friendly

**Font Weights:**
- **800 Bold** - Headings, hero text
- **700 Bold** - Subheadings
- **600 Semibold** - Button text, labels
- **500 Medium** - Body emphasis
- **400 Regular** - Body text

### ADHD-Friendly Typography Rules

1. **Line Height:** 1.6 for body text (1.4 minimum)
2. **Line Length:** 50-75 characters maximum
3. **Paragraph Spacing:** 1.5em between paragraphs
4. **Font Size:** Minimum 16px for body text
5. **Contrast:** Minimum 4.5:1 ratio
6. **Alignment:** Left-aligned (never justified)
7. **Letter Spacing:** Default (slightly increased for all-caps)

### Type Scale

```
Hero:        48px / 3rem    (Font weight: 800)
H1:          36px / 2.25rem (Font weight: 700)
H2:          30px / 1.875rem (Font weight: 700)
H3:          24px / 1.5rem  (Font weight: 600)
H4:          20px / 1.25rem (Font weight: 600)
Body Large:  18px / 1.125rem (Font weight: 400)
Body:        16px / 1rem    (Font weight: 400)
Small:       14px / 0.875rem (Font weight: 400)
Caption:     12px / 0.75rem (Font weight: 500)
```

## Voice & Tone

### Voice Attributes

Our voice is consistent across all content:

- **Clear & Concise** - Short sentences, no jargon
- **Encouraging** - Supportive without being patronizing
- **Honest** - Transparent about limitations
- **Inclusive** - Neurodiversity-affirming language

### Tone Variations

**Marketing/Website:**
- Confident but not arrogant
- Inspiring but not overpromising
- "Where focus takes flight" - aspirational

**Documentation:**
- Direct and instructional
- Step-by-step with clear examples
- Assume intelligence, not prior knowledge

**Error Messages:**
- Helpful and solution-oriented
- Never blame the user
- Provide clear next steps

**Community:**
- Warm and welcoming
- Patient and understanding
- Celebrate small wins

### Writing Guidelines

✅ **DO:**
- Use active voice
- Start with the main point
- Break content into scannable chunks
- Use bullet points and lists
- Include examples
- Write for screen readers

❌ **DON'T:**
- Use ableist language ("crazy," "insane," "OCD")
- Assume neurotypical patterns
- Create long, dense paragraphs
- Use idioms or unclear metaphors
- Write in all caps (except acronyms)

### Example Phrases

**Good:**
- "Tasks saved automatically"
- "3 of 5 steps complete"
- "This might take a few minutes"
- "Want help breaking this down?"

**Avoid:**
- "Don't worry, we've got you!"
- "It's easy!"
- "Just focus!"
- "Simply..."

## UI Components

### Buttons

**Primary Button:**
- Background: Blue Primary (#3B82F6)
- Text: White (#FFFFFF)
- Padding: 12px 24px
- Border radius: 8px
- Font weight: 600

**Secondary Button:**
- Background: White/10% opacity
- Border: 1px solid White/20%
- Text: White (#FFFFFF)
- Hover: White/20% opacity

**Danger Button:**
- Background: Red (#EF4444)
- Use sparingly for destructive actions

### Input Fields

- Background: White/10% opacity
- Border: 1px solid White/20%
- Focus: 2px Blue Primary ring
- Padding: 12px 16px
- Border radius: 8px

### Cards

- Background: White/5% opacity
- Border: 1px solid White/10%
- Padding: 24px
- Border radius: 16px
- Hover: Slight elevation

## Imagery & Graphics

### Photography

- **Style:** Clean, uncluttered backgrounds
- **Focus:** Tools, workspaces, hands working
- **Avoid:** Stock photos of overly happy people
- **Treatment:** Slight blue color grade to match brand

### Illustrations

- **Style:** Geometric, abstract
- **Colors:** From brand palette
- **Complexity:** Simple, not overwhelming
- **Purpose:** Supplement, don't distract

### Background Patterns

- Subtle grid patterns (3% opacity)
- Starfield/constellation patterns
- Abstract geometric shapes
- Always low contrast to not distract

## Accessibility Standards

### WCAG 2.1 Level AA Compliance

- Minimum contrast ratio: 4.5:1 (text)
- Minimum contrast ratio: 3:1 (UI components)
- All functionality available via keyboard
- Screen reader compatible
- No information conveyed by color alone

### ADHD-Specific Accommodations

- Reduced motion option
- Adjustable notification frequency
- Clear visual hierarchy
- Consistent navigation
- Undo for all destructive actions
- Auto-save (never lose work)

---

## Brand Applications

### Website

- Dark mode primary (matches design system)
- Light mode optional
- Consistent header/footer
- Clear navigation
- Prominent CTAs

### App Interface

- Offline-first messaging
- Clear loading states
- Helpful error messages
- Progressive disclosure
- Keyboard shortcuts visible

### Social Media

- Consistent profile images
- Brand colors in graphics
- Clear, readable text overlays
- Always include alt text

### Marketing Materials

- Lead with benefits, not features
- Include testimonials from ADHD users
- Show real interface screenshots
- Transparent about development status
